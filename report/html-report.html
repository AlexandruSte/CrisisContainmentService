1. Introduction

Crisis Center (Crisis Containment Service) is a Web platform which allows managing situations of crisis (for instance, earthquakes, floods, fire, tsunamis) by authorities or different organisations.

The application integrates Google Crisis Map and Fusion Tables for creating suplimentary layers with the purpose of providing information of interest regarding the events and the affected zones. Moreover, CAP (Common Alerting Protocol) will be used for notifying the population affected directly or in the nearby area. Additionally, social media platforms can be used. Using Geolocation API and services like Google Person Finder, the system can provide automatically information used for finding a missing person and regarding eventual shelters and saving routes.

The website will provide users with the ability to check again the past events in their area and if they were solved. Last but not least, users will be presented a variety of articles regarding how to help organisations, what to do in a crisis, first-aid, a page where world-wide current events are being shown and also a page where other organisations in partnership with Crisis Center can be found.

3. Database

In this application, we make use of a database in order to store all the information so that people can see and create alerts when they need to.

user and authority tables provide storage for all existing people on the platform.

alert table is used to store all the alerts made by users or authorities.

  • required : location, type, description
  • isSolved - 1 for "unresolved", 2 for "in progress", 3 for "done"
  • userType - 1 for Users, 2 for Authorities

missing_person table stores all the alerts made by users about a missing person.

  • required : firstName, lastName, lastInterraction, description
  • optional : photo
  • isSolved - 1 for "unresolved", 2 for "in progress", 3 for "done"
  • id_user refers to the user that created the alert

alert_creator table associates an alert with an user/authority.
